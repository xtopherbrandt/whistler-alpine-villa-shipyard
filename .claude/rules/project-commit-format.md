# Commit Message Format

This project uses Conventional Commits. Follow these conventions:

Format: `<type>(<scope>): <description>`
Case: lowercase
Scopes: ui, booking, api, auth, db, infra, config (add new scopes as the project grows)

Examples from this project:
- feat(ui): add property listing carousel
- fix(booking): handle date overlap validation
- chore: update dependencies
